MiddleOut - Data Flow & Error Handling

Complete pipeline from hotkey press to completion sound

Processing Pipeline
1. User presses Ctrl + Option + J
↓
2. HotkeyManager fires callback
↓
3. FinderBridge runs AppleScript → returns [URL]
↓ empty? → play error sound, no panel
4. FileRouter classifies each URL:
.heic .png .tiff .webp .jpg → ImageProcessor
.pdf → PDFProcessor
others → skip list
↓ all skipped? → show panel with "no supported files"
5. ProgressPanel appears → show total file count
↓
6. Process each file sequentially on background thread:
a. Validate file content (magic bytes via CGImageSource / PDFDocument)
→ invalid? skip with "corrupted or mismatched format"
b. Convert / compress → write to originalName_opt.jpg
→ write failed? skip with "permission denied" or "disk full"
c. Update progress bar on main thread
↓
7. All done → panel shows Done! + summary
↓
8. SoundPlayer plays completion chime
↓
9. Panel auto-dismisses after 2 seconds

Error Handling Strategy

Scenario Behavior
Finder not frontmost / no selection Play a short error sound. No panel appears.
All files are unsupported formats Panel shows "No supported files in selection" then auto-dismiss.
File extension faked (e.g. .docx renamed to .heic) CGImageSource/PDFDocument fails to open → skip with "corrupted or mismatched format"
File is locked / no read permission Skip with "cannot read file"
Output write fails (sandbox / disk full) Skip with "cannot write output"
Output file already exists (name_opt.jpg) Append number: name_opt_2.jpg, name_opt_3.jpg...
AppleScript permission denied Show one-time alert: "MiddleOut needs Automation permission for Finder" with button to open System Settings

Key Design Decisions

  • Sequential processing — one file at a time, simple and predictable progress
  • Background thread — all I/O on a dedicated queue, UI updates on main thread
  • Never crash, never block — every error is a "skip" with a human-readable reason
  • PDF output: creates folder filename_pages/ containing page_1.jpg, page_2.jpg...
  • Duplicate prevention: if _opt.jpg already exists, increment suffix
